# Prompt 02 - Domain Rules and Architecture

## Intent

Translate the requirement document into an enterprise-style domain model and a rules-driven configurator experience.

## Working prompt

Model markets, dealers, lifecycle paths, vehicle models, engines, transmissions, trims, finishes, wheels, and packages. Implement rule evaluation for dependencies, exclusions, pricing, compliance, and manufacturing feasibility. Make the UI respond step-by-step with operational summary, audit visibility, and recent quote persistence.

## Resulting direction

- catalog-driven option model
- centralized evaluator for corrections and business rules
- bootstrap data for UI initialization
- review/save workflow supporting quote and order paths
